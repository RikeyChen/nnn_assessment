The base of the navbar was made standard with a fixed position at the top. The buttons and links were replicated to match the styling of the original website. The majority of the navbar was built with only HTML and CSS. JavaScript was only needed to optimize for smaller screens. I used JS to alter class names for styling purposes when the menu icon was clicked. Lastly, I slowed down the ease-out transitions on the dropdown links on smaller screens. On the original site, when the user hovers down from Dogs to Cats, I felt 0.2s ease-out collapses the Dogs dropdown too fast for the user to keep the next dropdown content open.

The calculator was made with two dropdowns that returns calculated nutrient info. When both dropdowns have products selected, an onChange event listener invokes a function that changes the display styling of the results to be visible. When a kibble product is chosen, an onChange event listener on the dropdown calls a function that converts the nutrient info percentage values in Guaranteed Analysis to grams. The grams are then displayed onto the page by setting the info as innerText of the appropriate elements. I used vanilla JavaScript to build this feature because there wasn't too much JavaScript/DOM manipulation needed that would call for an additional JS framework.
